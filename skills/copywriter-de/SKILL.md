# Copywriter / Produkttexter (Deutsch)

## Prinzip

**Nutzen vor Feature. Konkret vor abstrakt. Kurz vor lang.**

Leser entscheiden in Sekunden ob ein Text relevant ist. Der erste Satz entscheidet. Kein Warm-up, keine Einleitung, keine Frage.

## Stimme

Kompetent ohne aufgeblasen zu sein. Direkt ohne unhöflich zu sein. Vertrauenswürdig statt werblich.

Der Text klingt wie jemand, der das Produkt kennt und es einem Kollegen empfiehlt — nicht wie ein Verkäufer.

## Rhythmus: Das wichtigste Werkzeug

Satzlänge variiert. Stark. Ein Satz kann drei Wörter haben. Der nächste kann eine Abhängigkeit oder eine Einschränkung erklären, die das Vorige in Kontext setzt. Dann wieder kurz.

Nie vier Sätze gleicher Länge hintereinander. Das ist das deutlichste AI-Muster überhaupt.

## Verbotene Wörter und Phrasen

**Hohlphrasen ohne Substanz:**
ganzheitlich, nachhaltig (ohne Beleg), innovativ, state-of-the-art, zukunftsorientiert, maßgeschneidert, reibungslos, nahtlos, game-changer, revolutionär, bahnbrechend, cutting-edge

**Typische AI-Satzanfänge — komplett verboten:**
- „In der heutigen Zeit…" / „In der modernen Welt…"
- „Sicherlich kennen Sie…" / „Kennen Sie das Gefühl…?"
- „Es ist wichtig zu erwähnen…"
- „Nicht zuletzt…" / „Zu guter Letzt…"
- „Außerdem", „Darüber hinaus", „Des Weiteren" als Satzanfang
- „Zweifelsohne", „Bekanntermaßen"

**Strukturmuster — verboten:**
- Em-Gedankenstrich (— oder --)
- „Nicht nur X, sondern auch Y" als Formel
- Falsche Konzession: „Obwohl X Einschränkungen hat, ist es dennoch bemerkenswert"
- Rhetorische Einstiegsfrage: „Was wäre wenn…?"
- Nummerierte Listen als Hauptstruktur: „5 Gründe warum…"
- Aufzählungen aus bloßen Substantiven ohne Verb
- Hedge-Stapel: „könnte möglicherweise", „dürfte eventuell"

## Produkttexte

Lead: Nutzen in einem Satz. Nicht „Das Plugin bietet X" sondern was der Nutzer damit erreicht.

```
❌ Das Import-Plugin bietet eine umfangreiche Schnittstelle zur Datenverarbeitung
   mit flexiblen Konfigurationsmöglichkeiten.

✅ 50.000 Produkte importieren, ohne die Datenbank zu belasten.
   Das Plugin verteilt die Last automatisch auf Warteschlangen.
```

Superlative nur mit Beleg. Keine Behauptung ohne Grundlage.

## Technische Blogbeiträge

**Erste Zeile = Kernaussage.** Nicht Kontext, nicht Einführung.

Sub-Headlines sind Aussagen, keine Themen:
```
❌ Performance
✅ Warum PHP 8.4 in diesem Benchmark dreimal schneller ist
```

Konkrete Codebeispiele schlagen abstrakte Erklärungen. Kein abschließender Abschnitt mit dem Titel „Fazit" — den Gedanken stattdessen im letzten Absatz natürlich zu Ende führen.

## Call-to-Action

Spezifisch, nicht generisch:
```
❌ Mehr erfahren   ❌ Jetzt entdecken
✅ Demo buchen     ✅ Plugin kostenlos testen
```

## Absatzlänge

Maximal 3–4 Sätze pro Absatz. Längere Blöcke wirken wie AI-Output und werden übersprungen.

## SEO-Pflichten

**REQUIRED SUB-SKILL:** Bei Blog- und Webseitentexten immer `seo` laden und einhalten.

Kurzfassung der wichtigsten Limits (Details im SEO-Skill):

| Element | Limit |
|---|---|
| Title-Tag | 50-60 Zeichen, Keyword vorne, ≤580 px |
| Meta-Description | 120-158 Zeichen, CTA enthalten |
| H1 | Genau eine pro Seite, enthält Primary Keyword |
| URL/Slug | 3-5 Wörter, Kleinbuchstaben, Bindestriche, Umlaute umschreiben |
| Absatz-Headlines (H2/H3) | Sequentiell verschachtelt, keine Ebenen überspringen |

Sub-Headlines sind Aussagen (siehe oben) UND enthalten semantisch relevante Keywords.

## Selbst-Check vor Abgabe

- [ ] Erster Satz nennt den Nutzen, keine Einleitung
- [ ] Kein em-Gedankenstrich im Text
- [ ] Kein Satz beginnt mit einem der verbotenen Übergänge
- [ ] Satzlänge variiert sichtbar — kurz und lang wechseln sich ab
- [ ] Keine Aufzählung mit bloßen Substantiven
- [ ] Jede Behauptung ist konkret oder belegt
- [ ] CTAs sind spezifisch
- [ ] Title-Tag ≤60 Zeichen, Primary Keyword vorne
- [ ] Meta-Description 120-158 Zeichen mit CTA
- [ ] Genau eine H1, Heading-Hierarchie eingehalten
- [ ] URL/Slug kurz, keyword-basiert, Umlaute als ae/oe/ue/ss
